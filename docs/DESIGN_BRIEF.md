# H.W. PDF TOOL — 디자인 브리프

> AI 디자인 도구(Stitch, Claude 등)에 입력하는 용도의 제품 설명서.
> 정보 구조·기능·문구는 그대로 유지하고, 시각 디자인은 자유롭게 새로 제안해도 됩니다.

## 1. 제품 한 줄 소개

PDF를 합치고, 정리하고, 변환하는 **오프라인 웹 도구**. 모든 처리가 사용자 PC의 브라우저 안에서만 일어나며 파일이 외부로 전송되지 않는다 (보안이 중요한 회사 환경용).

## 2. 사용자와 사용 환경

- 사용자: 사무직 직장인 1명 (개인 도구). 보험사에서 교육자료·보고서·교안을 다룬다
- 환경: 데스크톱 브라우저(크롬) 중심. 회사 PC는 인터넷이 차단될 수 있음 → 외부 리소스(CDN 폰트, 외부 이미지) 사용 불가, 모든 에셋은 파일 안에 내장
- 기술 형태: 단일 HTML 파일 (HTML + CSS + 바닐라 JS). 프레임워크 없음
- 모바일은 보조 — 반응형이면 충분

## 3. 전체 화면 구조

SPA 해시 라우팅. 화면은 **홈 1개 + 기능 화면 8개**.

```
홈 (도구 카드 그리드)
├── PDF 합치기
├── 페이지 정리
├── 워터마크 지우기
├── 이미지 → PDF
├── PDF → 이미지
├── PDF → PPT
├── 박스로 가리기      (예정 — 디자인에 포함해 주세요)
└── 글자·이미지 넣기   (예정 — 디자인에 포함해 주세요)
```

상단 내비게이션 바: 좌측 로고 "H.W. PDF TOOL", 기능 화면에서는 "← 홈으로" 버튼 표시.

## 4. 공통 인터랙션 패턴

모든 기능 화면은 같은 흐름을 따른다:

1. **업로드 존** — 점선 테두리의 큰 드롭 영역. 클릭 또는 드래그앤드롭. 아이콘 + "파일을 드래그하거나 클릭하여 선택" + 힌트 문구
2. **파일 목록 / 옵션** — 올린 파일명·용량 표시. 복수 파일이면 드래그로 순서변경, ✕로 제거
3. **실행 버튼** — 주황색 주 버튼 (예: "합치기", "변환하기")
4. **로딩 오버레이** — 전체 화면 반투명 + 스피너 + 진행 문구 "처리 중... (3/16)"
5. **결과** — 성공: 완료 카드 (파일은 자동 다운로드됨) / 실패: 빨간 에러 메시지 박스

## 5. 화면별 상세

### 5-1. 홈

- 히어로: 제목 "H.W. PDF TOOL" + 부제 "모든 작업이 이 PC 안에서만 처리됩니다 · 인터넷 불필요"
- 도구 카드 8개 그리드. 카드: 아이콘 + 이름 + 한 줄 설명
  - 📎 PDF 합치기 — 여러 PDF를 하나로
  - 🗂️ 페이지 정리 — 페이지를 보면서 삭제 · 순서변경 · 회전 · 뽑아내기
  - 🧽 워터마크 지우기 — 노트북LM 로고를 흔적 없이 (PDF · PPTX)
  - 🖼️ 이미지 → PDF — 사진 · 이미지를 PDF로 묶기
  - 📷 PDF → 이미지 — 페이지를 그림 파일로
  - 📊 PDF → PPT — 페이지를 슬라이드로
  - 🩹 박스로 가리기 — 보이면 안 되는 부분을 덮어서 저장 (준비중 배지)
  - ✍️ 글자·이미지 넣기 — 도장, 사인, 연락처를 원하는 자리에 (준비중 배지)

### 5-2. PDF 합치기

- 복수 PDF 업로드 → 파일 목록 (드래그 순서변경, 개별 삭제)
- 버튼 "합치기" (2개 미만이면 비활성)

### 5-3. 페이지 정리 (가장 복잡한 화면)

- PDF 1개 업로드 → 전체 페이지가 **썸네일 그리드**로 펼쳐짐
- 썸네일 카드: 페이지 미리보기 이미지 + 원본 페이지 번호 + 액션 버튼 2개 (🔄 90° 회전, ✕ 삭제)
- 썸네일을 드래그해서 순서 변경 (드롭 대상에 하이라이트)
- 문서 방향 자동 감지: 가로 문서는 넓은 칸, 세로 문서는 좁은 칸
- 상단 툴바: [이대로 저장] 주 버튼 + [처음으로 되돌리기] 보조 버튼 + 우측 "8페이지" 카운터

### 5-4. 워터마크 지우기

- PDF 또는 PPTX 1개 업로드 → 파일 표시 → 버튼 "워터마크 지우기"
- 설명: "노트북LM이 만든 PDF · PPTX 우하단의 NotebookLM 로고를 주변 배경색으로 자연스럽게 덮습니다."

### 5-5. 이미지 → PDF

- 복수 이미지(JPG·PNG·WEBP) 업로드 → **썸네일 그리드** (그림 미리보기 + 순번·용량 + ✕ 빼기, 드래그 순서변경) → 버튼 "PDF로 묶기"

### 5-6. PDF → 이미지

- PDF 1개 업로드 → 옵션 2개: 파일 형식 (PNG 선명함 / JPG 가벼움), 화질 (보통·좋음·매우 좋음) → 버튼 "변환하기"
- 여러 페이지면 ZIP으로 받음

### 5-7. PDF → PPT

- PDF 1개 업로드 → 옵션 1개: 화질 → 버튼 "변환하기"

### 5-8. 박스로 가리기 (예정 — 디자인 필요)

- PDF 업로드 → 페이지를 크게 보면서 마우스 드래그로 가릴 영역(사각형) 지정
- 색상 도구: 밝은 회색(기본) / 색상 선택기 / 스포이드(문서 클릭으로 색 추출) / 자동 배경색
- 좌우 페이지 이동, 지정한 박스 목록 관리(개별 삭제)
- 저장하면 가린 부분이 복구 불가능하게 평면화됨 — 이 점을 안내하는 문구 필요

### 5-9. 글자·이미지 넣기 (예정 — 디자인 필요)

- PDF 업로드 → 페이지 위에 글자(이름·연락처·날짜) 또는 이미지(도장·사인)를 드래그로 배치
- 자주 쓰는 항목 저장 목록 (브라우저 저장, 클릭 한 번으로 재사용)
- 크기 조절 핸들, 페이지 이동, 배치 항목 관리

## 6. 컴포넌트 목록

- 내비게이션 바 (로고, 뒤로가기)
- 도구 카드 (아이콘·제목·설명·준비중 배지)
- 업로드 드롭존 (기본 / 드래그오버 상태)
- 파일 아이템 (이름·용량·삭제 버튼·드래그 핸들)
- 썸네일 카드 (미리보기·페이지번호·회전/삭제 버튼·드래그 상태)
- 주 버튼 / 보조(고스트) 버튼 / 비활성 상태
- 셀렉트 박스, 텍스트 입력, 라디오 그룹
- 로딩 오버레이 (스피너 + 진행 문구)
- 결과 카드 / 에러 메시지 박스

## 7. 현재 디자인 (참고용 — 더 좋은 제안 환영)

- 배경: 웜 그레이(#f5f5f4) + 주황 계열 라디얼 글로우 장식
- 포인트 컬러: 주황 #f97316
- 글래스모피즘: 반투명 화이트 카드 + 블러
- 폰트: Pretendard (시스템 폰트 폴백 필수 — 오프라인 환경)
- 라운드 16px, 부드러운 그림자, 호버 시 살짝 떠오름

## 8. 디자인 시 지켜야 할 것

- 모든 문구는 위에 적힌 한국어 그대로 사용
- "파일이 PC 밖으로 나가지 않는다"는 신뢰감이 첫인상에서 느껴질 것
- 코딩 초보가 유지보수하므로 과한 애니메이션·복잡한 레이아웃보다 단순하고 명확한 구조 선호
- 외부 이미지·외부 폰트 URL 사용 금지 (오프라인에서 깨짐)
